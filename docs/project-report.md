# Project Report – Neuronales Netz

## Overview
- **Goal:** Build a feed-forward digit classifier and compare different training datasets, augmentation pipelines, and models while keeping everything reproducible on local hardware.
- **Key deliverables:** dataset loader improvements, fully documented training flows, multiple checkpoints (`models/initial_model.npz`, `models/fedora_chip_test_1.npz`, `models/fedora_chip_test_2.npz`, `models/kaggle_mnist_full.npz`), automated exports, and a tested Tkinter app.

## Datasets & Preprocessing
| Dataset | Description | Entries | Role |
|---------|-------------|---------|------|
| `data/Reduced_MNIST_Data` | Cropped/reduced MNIST for quick prototype training | ~1k | Baseline & experimentation, includes train/test splits that ran the early experiments.
| `data/kaggle_mnist/mnist_png` | Standard 60k/10k PNG MNIST | 70k | Source for the high-accuracy checkpoint; loader now handles PNG normalization and grayscale conversion (`data_loader.load_images_from_folder`).
| `datacreation` Postgres tables | Live market metrics, price snapshots, wipe events from Hyperliquid | 6+ million records (grows) | Surface insights for future AI-driven strategies via `scripts/export_all_tables.py`.

## Models & Experiments
| Checkpoint | Dataset | Epochs | Val accuracy | Test accuracy | Notes |
|------------|---------|--------|--------------|---------------|-------|
| `models/initial_model.npz` | Reduced MNIST (default) | 80 | ~0.89 | ~0.91 | Legacy baseline shipped with repo.
| `models/fedora_chip_test_1.npz` | Reduced MNIST + validation split | 72 | 0.925 | 0.941 | Early deeper architecture + augmentation.
| `models/fedora_chip_test_2.npz` | Reduced testing folder only | 52 | 0.900 | 0.874 | Demonstrates dataset ceiling when training/test overlap.
| `models/kaggle_mnist_full.npz` | Full Kaggle PNG MNIST | 75 | 0.9705 | 0.9736 | Best model; uses PNG-aware loader + LR schedule.

## Reproducibility
1. Create a fresh environment: `make install`. It bootstraps `.venv` and installs dependencies.
2. To copy the best model, run `make train-kaggle` (calls `main.py --train` with `data/kaggle_mnist/mnist_png`).
3. Validate the loader via `make test-loader` (runs `tests/test_loader.py`).
4. Generate metric visualizations with `make plot-metrics` (creates `docs/figures/accuracy_comparison.png`).

## Results & Visuals
See `docs/figures/accuracy_comparison.png` (generated by `scripts/plot_metrics.py`) for a side-by-side of validation/test curves across checkpoints.

## Lessons Learned
- Normalization matters: once the PNG loader normalized per-image max values and clamped values to [0,1], the Kaggle dataset jumped from random-guessing to 97% accuracy.
- Bigger datasets + careful augmentation + LR decay beat deeper models trained on small data.
- Building automation (Makefile + scripts) makes retraining reproducible under tight deadlines.
